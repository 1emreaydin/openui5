Manual Integration Test: OData Products App

DESCRIPTION

Test for the sap.ui.model.odata.type.Unit and sap.ui.model.odata.type.Currency types

PREPARATION

To check console errors, open the developer tools before starting the application and check with each step that there are no errors and warnings.

Testing with mock data

Start the Products application with the following URL (adjust host and port if necessary):
  • local: http://localhost:8080/testsuite/test-resources/sap/ui/core/demokit/sample/common/index.html?component=odata.v4.Products

Testing with backend

Configure your IDE to be able to execute the Products application against an ABAP backend system.
Start the Products application with the URL (adjust host and port if necessary):
  • local: http://localhost:8080/testsuite/test-resources/sap/ui/core/demokit/sample/common/index.html?component=odata.v4.Products&realOData=true

REALIZE and CHECK

Check whether measures are handled properly regarding decimal places defined for the corresponding unit code

Testing with mock data

For testing, you may use the following valid unit codes:
  • KG and G (0 decimal places)
  • % (0 decimal places)
  • DEG (1 decimal places)
  • NO (2 decimal places)
  • M/L (3 decimal places)
  • μG (6 decimal places)
For a complete list of valid unit codes, see the "ExternalCode" property of the objects in http://localhost:8080/testsuite/test-resources/sap/ui/core/demokit/sample/odata/v4/Products/data/UnitsOfMeasure.json

Testing with backend

For testing, you may use the unit codes listed in "Testing with mock data". Note: Although %, DEG, NO and M/L are valid, the backend does not accept them as they are no weight units; you see a corresponding error message "Invalid weight unit..." when saving a weight with one of these units upon leaving the weight input field. For a complete list of valid unit codes, see the "ExternalCode" property of the objects in http://localhost:8080/testsuite/proxy/sap/opu/odata4/sap/zui5_testv4/default/iwbep/common/0001/UnitsOfMeasure?$select=ExternalCode,DecimalPlaces,Text

Test measures

  • Check: Weights in the table are displayed with the number of decimal places associated with the corresponding unit.
  • Change the weight for a product in the table using a valid unit code having 3 or less decimal places such as 'DEG' and leave the input field.
  • Check: The weight measure is formatted with the decimal places associated with the entered unit code.
  • Change the weight unit to an invalid code such as 'XYZ'.
  • Check: You see an error message and the input field shows an invalid data state.
  • Change the weight measure to a number with more than three decimal places.
  • Check: You see a validation error message and the measure field shows an invalid data state.

Check whether currencies are handled properly regarding decimal places defined for the corresponding currency code

Testing with mock data

Execute the test with "EUR3" which has 3 decimal places in case mock data is used. In addition, you may use the following valid currency codes for testing:
  • JPY (0 decimal places)
  • EUR and USD (2 decimal places)
  • BHD (3 decimal places)
For a complete list of valid currency codes, see the "CurrencyCode" property of the objects in http://localhost:8080/testsuite/test-resources/sap/ui/core/demokit/sample/odata/v4/Products/data/Currencies.json

Testing with backend

For testing, you may use the currency codes listed in "Testing with mock data". http://localhost:8080/testsuite/proxy/sap/opu/odata4/sap/zui5_testv4/default/iwbep/common/0001/Currencies

Test currencies

  • Check: Prices in the table are displayed with the number of decimal places associated with the corresponding currency.
  • Change the price for a product in the table using a valid currency code and leave the input field.
  • Check: The price is formatted with the decimal places associated with the entered unit code.
  • Change the currency unit to an invalid code such as 'XYZ'.
  • Check: You see an error message and the price field shows an invalid data state.
