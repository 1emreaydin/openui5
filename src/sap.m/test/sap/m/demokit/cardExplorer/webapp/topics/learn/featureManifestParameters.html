Manifest Parameters

Overview

Cards can define input parameters in their manifests. The input parameters can be everything including user IDs, authentication tokens, locations, language etc... It is possible to assign default values to the input parameters in the manifest. When creating or integrating a card, it is possible to assign values to parameters otherwise they are replaced with their default values. Parameters are described in the configuration section of the manifest. After that they can be used with a double bracket syntax like: "{{parameters.city}}". All parameters are replaced during manifest processing and changing them will cause re-processing of the manifest.

Examples

A card having a city parameter with a default value of "Walldorf" which is used inside the request:

{
	"sap.card": {
		"configuration": {
			"parameters": {
				"city": "Walldorf"
			}
		},
		"type": "List",
		"header": {
			"title": "Weather"
		},
		"content": {
			"data": {
				"request": {
					"url": "weather/api?q={{parameters.city}}"
				}
			},
			"item": {
				"icon": {
					"src": "{/current/condition/icon}"
				},
				"title": {
					"value": "{/location/name}"
				},
				"description": {
					"value": "{/current/condition/text}"
				}
			}
		}
	}
}

Changing the city parameter can be done by using the API of the card. This will cause manifest re-processing: 

var oParameters = {
	"city": "Sofia"
}
oCard.setParameters(oParameters);
